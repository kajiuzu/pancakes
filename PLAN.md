# フライングパンケーキ - 実装計画（Playable v1）

## ゲームの目的
奥のシェフから飛んでくるパンケーキを、お皿を左右に動かしてキャッチし、できるだけ高く積み上げる。最終的な「高さ（段数）」と「スコア」を競う。

## プレイヤー操作
- `←` / `→` または `A` / `D`: お皿を左右移動
- `Space`: ゲーム開始 / リトライ

## メインループ
1. 入力処理（お皿の移動）
2. 次のパンケーキ生成（左上方向から放物線で飛来）
3. パンケーキ位置更新（重力で落下）
4. 皿・塔との衝突判定
5. 積み上げ時のズレ量（オフセット）を計算
6. バランス値更新（グラグラ度）
7. UI更新（高さ/スコア/安定度/ミッション）
8. 勝敗判定
9. 描画

## 勝利条件
- エンドレス前提のため、明示的勝利は「自己ベスト更新」。
- v1では **20段到達** で「CLEAR」表示を出す（そのまま継続可）。

## 失敗条件
- パンケーキをキャッチできず地面に落とす（Miss）
- 積み上げ時のズレが大きすぎて塔が崩壊判定になる
- バランスゲージが0になる

## 難易度の上がり方
- 段数が増えるほど
  - 飛来速度アップ
  - 厚いパンケーキ出現率アップ（高得点だが不安定）
  - 許容オフセット縮小

## 画面構成
- 上部HUD
  - SCORE
  - HEIGHT（段数）
  - STABILITY（安定度ゲージ）
  - MISSION（簡易ミッション）
- 中央ゲームエリア（Canvas）
  - 左奥シェフ位置（発射点の演出）
  - 皿（プレイヤー）
  - 積み上がるパンケーキ塔
- 下部メッセージ
  - 操作説明 / Game Over / Clear

## 必要なアセット
v1は外部アセットなし（図形描画）
- パンケーキ（厚さ2種: thin / thick）
- 皿
- 背景グラデーション
- ミッション通知UI

## 技術スタック
- Next.js（App Router）
- Phaser 3（レンダリング/入力/ループ）
- Fastify/Postgres/Redis/OpenAI API は将来拡張時に採用

## 実装順序
1. 企画要件に合わせてルール再定義（飛来→キャッチ→積み）
2. HUD項目を「Score / Height / Stability / Mission」に変更
3. 物理簡易モデル（放物線飛来 + 着地判定）を実装
4. 積みズレと崩壊ロジック実装
5. 厚さ違い・スコア差・難易度スケール実装
6. ミッション（例: 右側3連続着地）実装
7. ローカル起動して動作確認
